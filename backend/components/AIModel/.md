AI Model (Python)
=================

Purpose
-------
Train a production-grade tabular classifier that predicts heat risk level from weather
and heat index data, then generates an advisory payload compatible with the backend `AdvisoryResult`.

Data used
---------
- `public.weather_data` (temperature, humidity, wind, pressure)
- `public.heat_index_logs` (heat index + heat_level label)

Model output
------------
- `riskLevel`: one of `safe`, `caution`, `extreme-caution`, `danger`, `extreme-danger`
- `summary`, `actions`, `safetyTips` based on templates
- `decisionBasis` mirrors backend schema

Training notes
--------------
- Model: XGBoost multi-class classifier
- Split: time-based (older data for training, newest data for test)
- Calibration: isotonic probability calibration for confidence scores
- Class imbalance: balanced sample weights
- Guardrail: optional minimum per-class count gate (skip training if any label is under the threshold)

Quick start
-----------
1) Install dependencies (from this folder):

  `pip install -r requirements.txt`

Or from the repo root on Windows:

  `backend\scripts\setup-python-venv.ps1`

2) Train the model (uses DATABASE_URL from `.env` or your shell):

  `python ai.py train --env-file ../../.env --output-dir model`

Optional: skip training unless each risk level has enough rows:

  `python ai.py train --env-file ../../.env --output-dir model --min-class-count 50`

3) Predict from a JSON input:

	`python ai.py predict --model-dir model --input-json sample_input.json --language tagalog`

Training audit and release notes
--------------------------------
- `backend/scripts/audit-training-data.js` writes a quality report for label balance, parse failures, and drift.
- `backend/scripts/package-model-artifacts.ps1` copies release files into `backend/deploy/model-release/` and creates a zip.
- `model_registry.json` records promotion state, metrics, and artifact names after each retrain.

Sample input JSON
-----------------
```
{
  "temperatureC": 33.4,
  "humidityPercent": 68,
  "windSpeedMps": 2.8,
  "pressureHpa": 1007.2,
  "heatIndexC": 41.2,
  "source": "openweathermap"
}
```

Weekly retrain schedule
-----------------------
Use the helper scripts below with your scheduler of choice. These call the training job with
the minimum class-count gate so it safely skips until data is sufficient.

Linux cron (every Sunday at 2:00 AM):

```bash
0 2 * * 0 /path/to/backend/scripts/retrain-ai.sh 50 >> /var/log/beat-the-heat-retrain.log 2>&1
```

Windows Task Scheduler (PowerShell action):

```powershell
powershell -ExecutionPolicy Bypass -File "C:\path\to\backend\scripts\retrain-ai.ps1" -MinClassCount 50
```

GitHub Actions schedule
-----------------------
The repo includes `.github/workflows/retrain-ai.yml` for weekly retraining, auditing, and packaging.
